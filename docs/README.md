# OctoAcme Project Management Docs

This README provides a short summary of OctoAcme's project management processes and direct links to the canonical process documents in this repository's docs/ folder.

## Summary

OctoAcme runs cross-functional projects with a focus on **iterative delivery**, **clear ownership** (PM + Product Lead), **data-informed decisions**, and **continuous improvement**. 

### Key Practices

- **Initiation**: Structured entry point with problem statement, stakeholder alignment, and success metrics (one-pager)
- **Planning**: Prioritized backlog with acceptance criteria, Definition of Done, release plan, and risk identification
- **Execution**: Project board workflow, small PR conventions, CI/CD enforcement, quality gates (testing and security scanning)
- **Delivery**: Staged deployments with rollback plans and post-deployment verification
- **Retrospectives**: Regular capture of learnings and actionable improvements

### Core Roles

- **Project Manager (PM)**: Coordinates delivery, schedules, risks, and communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes backlog, and measures success
- **Developers**: Implement features, collaborate on design, write tests, and identify technical risks
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs, approvals, and business context

## Quick Start

1. **New project?** Start with the [Project Initiation Guide](octoacme-project-initiation.md) to validate the problem and align stakeholders.
2. **Ready to build?** Move to [Project Planning](octoacme-project-planning.md) to create your backlog and release plan.
3. **In flight?** Use [Execution & Tracking](octoacme-execution-and-tracking.md) for daily team rhythm and escalation guidance.
4. **Ready to ship?** Follow the [Release & Deployment Guide](octoacme-release-and-deployment.md).
5. **Finished or blocked?** Run a [Retrospective](octoacme-retrospective-and-continuous-improvement.md) to capture improvements.
